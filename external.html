This guide will walk you through the steps required to set up an external server and connect it to the Minehut network.

This documentation is designed to help you understand each step of the setup process. Please follow the instructions carefully. Basic server management knowledge is required. For more information, you can refer to Minecraft Wiki and PaperMC Documentation.

Requirements

To set up an external server, you will need the following:

  1. Server Hosting Provider
  2. Basic server management knowledge
  3. Access to your server’s configuration files

Be cautious when configuring your external server. Incorrect settings may cause connectivity issues or other problems.
Minehut support may not be able to assist with configuration issues.

External documentation

Setting up an external server involves several steps to ensure proper connectivity with Minehut. Follow the instructions below to get started:

External Server Setup

To set up an external server, follow these steps:

Step 1: Create Your External Server

Choose an external hosting provider and create a server. Configuration details will vary based on your host.

Step 2: Configure Minehut Player Authentication

Set up your server to use Minehut’s authentication service. The configuration will depend on your server type:

PaperMC Configuration

To use PaperMC with Minehut authentication, add the following flags to your server start command:

                        
                        java -Dmojang.sessionserver=https://api.minehut.com/mitm/proxy/session/minecraft/hasJoined -jar paper.jar

Step 3: Enable Proxy Protocol

Note: Skip this step if you’re using TCPShield for DDoS protection.

Enable proxy protocol in your proxy’s configuration file:

Velocity
                        
                        haproxy-protocol = true
Waterfall
                        
                        proxy_protocol: true
Paper
                        
                        proxy-protocol: true

Connect Your External Server

First, make sure you’ve set up your server plan on Minehut and configured your external server.

Step 1: Create a Minehut Account

Go to Minehut and create an account. You’ll use this account to manage your server.

Step 2: Set Your Server Plan to “External”

  1. Go to your server (or create a new one).
  2. Activate your server and open the Server Panel by clicking “Manage.”
  3. From your Server Panel, edit your server plan.
  4. Set your server plan to “External” and press confirm.
    Note: You’ll need to buy credits to use an external server and keep it active. The easiest way to ensure you have enough credits every month is to purchase a credit subscription.
  5. You’re now ready to add your external server.

Step 3: Complete External Server Settings

On Minehut, open your server’s server panel and navigate to the “External Server Settings” section.

Complete all the fields:

  • IP: Enter the public IP of your external server.
    • Your host will normally provide you with the public IP, which typically consists of four numbers separated by dots (e.g., `123.45.67.89`). This information is usually visible somewhere within the server panel.
    • A domain name can also be used here if you have one (e.g., play.myserver.com configured as an A record pointing to `123.45.67.89`).
    • If you are using TCP Shield, this will instead be your DDoS-protected IP.
  • Proxy Type: Select the type of proxy you’re using.
    • This will be “Velocity” or “Waterfall” if you are using either.
    • If using anything else (e.g., a standalone Paper server), select “Other.”
  • DNS Record Type: Select the type of record used for your server.
    • Use “Port” if you are using a public IP address or DDoS-protected IP.
    • Use “Port” if you are using a domain that does not use an SRV record.
    • Use “SRV” only if you use an SRV record on your server. If you are not using an SRV record or have a misconfigured SRV record, selecting this will break your server.
  • Port: The port used to connect to your external server.
    • The default Minecraft port is “25565,” but many hosts will assign you a different port, so you should check to make sure. Normally the port is displayed in the same place as the IP.
    • Players will not be able to connect if the port is wrong.
  • TCP Shield: Select whether you have configured TCP Shield for DDoS protection.
    • Most servers should leave this as “Not Configured” as most hosts already have built-in DDoS protection.
    • Select “Configured” only if you have configured your external server on TCPShield.com, or this will break your server.
